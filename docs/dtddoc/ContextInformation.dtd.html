Measurable Context Information


<contextInformation> Root element

A list of context information that will be made available through the interfaces of the rule generator. It should contain all the context information that can be acquired or processed by the context detection through widgets and interpreters.

<contextInformation>'s children
NameCardinality
informationAny number
Element's model:

(information)*


<information> Child of contextInformation

A context information that can be processed by or is available through widgets and interpreters.

<information>'s children
NameCardinality
contextClassesOne or none
contextValueOnly one
parametersOne or none
<information>'s attributes
NameValuesDefault
idMatch the ID rules.
multiplicityUNIQUE, MULTIPLEUNIQUE
nohistorynohistory
Element's model:

(contextClasses?, contextValue, parameters?)


@id Attribute of information

The identifier of the context information. This value should be used by widgets and interpreters to state that they provide or process the context information.

Possible values: Match the ID rules. - Required


@multiplicity Attribute of information

State if the context information is unique (only one value can exist simultaneously but a history of past values can be saved) or if multiple values can be present at the same time (this effectively deactivates the history for this information).

Possible values: UNIQUE, MULTIPLE - Default value: UNIQUE


@nohistory Attribute of information

States that no history will be available for this context information. This parameter is only relevant for unique context information. Context information with multiple values never saves old values.

Default value: nohistory
